Help

Sales Planning has a visual and flexible model in order to boost the sales management of your company. It has two main entities: Nodes and their Attributes and two main relations: Attributes for each Level and Nodes and their Attributes.

  • Nodes: Stand for any kind of group of information you may want to design and manage in a visual way (eg. Country, Industry, Stakeholders, etc). This version has 4 level of Nodes. Each level can be linked with levels immediately higher or lower, creating an hierarchy of parents and their childs. All possible associations:

  • Node Level 1 (higher): can be parent of Node Level 3
  • Node Level 2: can be child of Node Level 1 and parent of Node Level 2
  • Node Level 3: can be child of Node Level 2 and parent of Node Level 4
  • Node Level 4 (lower): can be child of Node Level 3

  • Attributes: Stand for any unit of information you need to manage (eg. budget, resposible, sales target, etc. All Attributes are created independently. After, they can be associated with Levels and finally with Nodes themselves, where are stored the actual information you provide.

  • Association between Attributes and Levels: Metadata association between attributes and each level. You must do this association in order to use the Attributes with actual Nodes.

  • Association between Nodes and Attributes: Actual information you store in the Attributes when you create a Node.

This current version has 3 pages as follows:

  • Main page (Nodes management): Create actual Nodes, make association among them and register actual information in attributes of each Node.

  • Attributes Registration: Manage all attributes (create, update and remove).

  • Node Level & Attributes Association: Associate each attribute among the 4 levels of Nodes.

Version Note: although the model and database has entities related to People and Teams (and associations), this version don't provide functionalities to manage this entities. It will be provided in the next version of Sales Planning.
